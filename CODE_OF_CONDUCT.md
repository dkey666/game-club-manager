# Code of Conduct

## Our standard

We want this project to be a respectful, practical, and technically serious open-source space. Contributors are expected to communicate clearly, assume good intent, and focus discussion on improving the project.

Examples of behavior that support a healthy community include:

- giving constructive and specific technical feedback
- being respectful in disagreement
- welcoming first-time contributors and helping them get unstuck
- sharing context, reproduction steps, and evidence when reporting problems
- accepting that maintainers may prioritize stability, security, and scope

Examples of unacceptable behavior include:

- harassment, personal attacks, or insulting language
- bad-faith arguments, trolling, or deliberate disruption
- publishing private information without permission
- spam, off-topic promotion, or manipulative behavior
- pressuring maintainers for support outside the repository workflow

## Scope

This Code of Conduct applies to repository discussions, issues, pull requests, reviews, and other project-related interactions.

## Enforcement

Project maintainers may remove, edit, or reject comments, issues, pull requests, or other contributions that violate this Code of Conduct. Repeated or severe violations may result in temporary or permanent restrictions from participating in the project.

## Reporting

For conduct concerns, contact the project maintainer privately. For security issues, use the process described in [SECURITY.md](SECURITY.md).
